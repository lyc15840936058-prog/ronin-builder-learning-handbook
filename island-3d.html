👁️平面屏幕怎么"骗"出立体感

先问个根本问题:你的屏幕明明是平的,凭什么能让你看到一个"立体"的方块?答案是——它在骗你的眼睛,而且用的是你大脑本来就吃这一套的把戏。

🔍 深挖：立体感是大脑"脑补"出来的

真实世界里,你判断一个东西"立体、有远近",靠的是几条线索:近大远小、近处挡住远处、光照产生的明暗、平行线向远处汇聚。

3D 程序做的事,就是在平面上精确地伪造这几条线索:把该小的画小、该被挡的挡住、该暗的地方画暗。你的大脑一看到这些熟悉的线索,就自动"脑补"出立体——根本不需要屏幕真的凸起来。

所以"3D 渲染"的本质,是一道数学题:已知一个虚拟立体世界、和一只虚拟眼睛(相机)的位置,算出"这只眼睛应该看到一张什么样的平面图",再把这张图画到你屏幕上。立体是假的,但骗得足够真。

🎨
就像画家画素描。
纸是平的,但画家用近大远小、明暗阴影,就让你看出一个立体的苹果。3D 程序是"会自动算明暗远近的超级画家"——你给它摆好物体和灯,它每一帧都帮你画出正确的素描。

下面这个方块,就是程序实时"画"出来的立体——你可以用鼠标拖动它,感受一下这个"骗局"有多真:

🧊 你的第一个 3D 方块用鼠标拖动它 / 滚轮缩放
↑ 它其实是画在平面上的。拖动时"近大远小、明暗变化"都在实时重算——这就是 3D 渲染。

🔍 深挖：为什么是这五样,各自不可替代

🌌 场景=一个空的虚拟空间,你把所有东西(物体、灯、相机)都放进它里面。它是"舞台"。

📷 相机=虚拟的眼睛。同一个场景,相机摆的位置/角度不同,你看到的画面就完全不同。"你看到什么"由相机决定。

🧊 物体=场景里真正的"东西"——一个方块、一个球、一个模型。它由"形状"和"材质"两部分组成(下面两条)。

🎨 材质=物体表面的"皮肤":什么颜色?像金属还是像塑料?光滑还是粗糙?同一个形状,换材质就完全不同质感。

💡 光=照亮场景的灯。这是新手最容易忘、又最致命的一样——下面第4站专门讲:很多人代码全对,屏幕却一片黑,就是忘了加灯。

打开灯💡,再用相机📷这只眼睛去看——一帧 3D 画面就成了。

📐物体是怎么"长"出来的:顶点 → 网格 → 面

一个 3D 物体的"形状",在电脑里到底是什么?答案可能出乎你意料:再复杂的 3D 模型,本质都是一大堆三角形拼成的。

🔍 深挖：从点,到线,到面,到形状

① 顶点(Vertex):3D 空间里的一个点,用三个数字表示它的位置——(x, y, z),分别是"左右、上下、前后"。这是 3D 世界最小的砖。

② 面(Face):三个顶点连起来,围成一个三角形。三角形是最简单的"平面片"——为什么用三角形?因为三个点必定在同一个平面上,计算最简单、最不会出错。

③ 网格(Mesh):成百上千个三角形拼在一起,就组成了一个物体的外壳,这层壳就叫"网格"。一个球是很多小三角形拼出来的"近似球";一个游戏角色是几万个三角形拼出来的。

所以你在游戏里看到的任何东西——人、车、山——剥开它的"皮肤",底下都是一张三角形织成的网。三角形越多,模型越精细,但也越吃性能。

⚽
就像足球表面的那些块、或低多边形(low-poly)的折纸风模型。
凑近看,曲面其实是一小片一小片平面拼的。三角形够多够小时,你的眼睛就看成了光滑的曲面。3D 物体全是这么"拼"出来的。

看下面这个球——左边正常显示,你点"看骨架"按钮,就能看到它其实是无数三角形拼的:

⚽ 一个球的"皮"与"骨"拖动旋转
💡 关键词记一下:三角形的数量叫"面数 / 多边形数"。你以后听到"这个模型多边形太高,太卡了",意思就是三角形太多、电脑算不过来。3D 的性能优化,很大程度就是在"形状够精细"和"三角形别太多"之间找平衡。

💡为什么没有光,你什么都看不见

这是新手第一大坑,值得单独一站讲透:在 3D 世界里,光不是"锦上添花",而是"能不能看见"的前提。

🔍 深挖：因为立体感本身就来自明暗

回到第1站:你的大脑靠明暗来判断立体。一个球之所以看起来圆,是因为它朝光的一面亮、背光的一面暗,中间有渐变——这个渐变骗大脑说"这是个球"。

如果场景里没有光,物体表面就没有任何明暗差异,要么整体一个死板的颜色(看着像贴纸,毫无立体感),要么(用了需要光的材质时)直接全黑、什么都看不到。

所以光在 3D 里干两件事:① 让你能看见物体;② 通过明暗,让物体显出立体。没有光,3D 就塌回了平面。

🎞️怎么让它动起来:渲染 + 动画循环

到现在我们说的都是"画出一帧"。但 3D 通常是动的——会转、会交互。它怎么动起来的?原理简单得让你意外:

🔍 深挖：动画 = 飞快地一帧帧重画

渲染(Render)=程序根据"当前场景+相机",算出并画出一帧画面的动作。一次渲染 = 一张静止的图。

让它动,靠的是一个动画循环(loop):程序每秒重复几十次这个过程——稍微挪一下物体的角度 → 渲染一帧 → 再挪一点 → 再渲染一帧……

因为快到每秒 60 次,你的眼睛把这一连串静止画面看成了连续的运动——和动画片、电影"一秒很多张"的原理一模一样。所以"让方块转起来"的本质是:在循环里,每一帧都把它的角度加一点点。

📖
就是翻页动画(手翻书)。
每一页画一个略微不同的姿势,飞快翻动,小人就"动"起来了。3D 的动画循环,就是电脑每秒帮你翻 60 页,每页都重新算好明暗远近。
🎞️ "让方块转起来"的核心,就这几行(伪代码)
function 每一帧() {
  方块.角度 = 方块.角度 + 0.01;   ← 每帧多转一点点
  渲染(场景, 相机);              ← 把这一帧画出来
  请求下一帧(每一帧);            ← 循环:马上再来一次
}
每一帧();                        ← 启动循环
读法:“每一帧:把角度加一点、画出来、然后请求马上再来一次。” 这个"请求下一帧"会被浏览器以约每秒60次的节奏调用——于是方块就匀速转起来了。所有 3D 动画,内核都是这个循环。

🛠️实战:Three.js 第一个方块(逐行讲)

原理通了,该动手了。网页里做 3D,最主流的工具叫 Three.js——它把上面那些数学和五件套都打包成了好用的积木,你不用自己算明暗远近。我们用它把前面学的五件套拼出来。

🔍 深挖：为什么用 Three.js,而不是从零写

浏览器底层画 3D 的能力叫 WebGL,但它极其原始难用(要自己写大量数学)。Three.js 是架在 WebGL 上的一层"友好积木"——你说"给我一个方块、一盏灯、一个相机",它替你把底层的脏活全干了。

这正是手册一贯的思路:不重复造轮子,站在现成工具的肩膀上。Three.js 是这个领域最成熟、资料最多、AI 最熟的选择,新手首选它。

下面是一个完整能跑的最小例子。看着长,其实就是"五件套"一件件摆出来。逐行注释了:

🧊 Three.js 最小完整例子(五件套对照看)
// 引入 three.js 工具库
import * as THREE from 'three';

// ① 场景🌌:一个空舞台
const scene = new THREE.Scene();

// ② 相机📷:那只眼睛(参数大致是:视野角度、画面宽高比、能看多近、多远)
const camera = new THREE.PerspectiveCamera(75, 16/9, 0.1, 1000);
camera.position.z = 3;          ← 把眼睛往后退3步,才看得到方块

// 渲染器:负责把画面真正画到网页上
const renderer = new THREE.WebGLRenderer();
renderer.setSize(600, 340);
document.body.appendChild(renderer.domElement);  ← 把画布放进网页

// ③ 物体🧊 = 形状(网格几何体) + ④ 材质🎨(皮肤)
const geometry = new THREE.BoxGeometry(1, 1, 1);      ← 一个1×1×1的方块形状
const material = new THREE.MeshStandardMaterial({ color: 0x6c8ce8 }); ← 蓝色、会反光的皮肤
const cube = new THREE.Mesh(geometry, material);   ← 形状+皮肤=物体
scene.add(cube);                    ← 把方块放进场景

// ⑤ 光💡:不加这个,方块会是黑的!
const light = new THREE.DirectionalLight(0xffffff, 1);  ← 一盏白色平行光
light.position.set(2, 2, 3);
scene.add(light);
scene.add(new THREE.AmbientLight(0xffffff, 0.4)); ← 再加点环境光,别全黑

// 🎞️ 动画循环:每帧转一点 + 渲染
function animate() {
  cube.rotation.x += 0.01;
  cube.rotation.y += 0.01;
  renderer.render(scene, camera);
  requestAnimationFrame(animate);  ← 请求下一帧,形成循环
}
animate();
别被长度吓到。从上到下,就是五件套依次登场:场景→相机→(渲染器)→物体+材质→光→动画循环。你不用记这些函数名,看懂"哪一段在干五件套里的哪件事"就够了——以后让 AI 写,你能看懂、能改、能验证,这就是 AI native 需要的能力。
💡 上面那个 0x6c8ce8 是什么:这是颜色的"十六进制"写法,和网页 CSS 里的 #6c8ce8 是同一个蓝色,只是 3D 里习惯用 0x 开头。看到别懵。

🖱️ 加鼠标拖动(最实用)

🔍 深挖：鼠标拖动转视角,本质是"动相机"不是动物体

你拖动鼠标"转动"物体时,其实物体没动——是相机📷绕着物体转了(回想第2站:你看到什么由相机决定)。让相机随鼠标绕物体转的功能很常用,Three.js 有现成的一个叫 OrbitControls(轨道控制器),加几行就能让用户自由拖看、滚轮缩放。

这就是为什么网页上的"3D 产品展示"你能360°拖着看——背后就是 OrbitControls 在让相机绕着产品转。

🖱️ 加上鼠标拖动,就这几行
import { OrbitControls } from 'three/examples/jsm/controls/OrbitControls.js';

const controls = new OrbitControls(camera, renderer.domElement);
controls.enableDamping = true;   ← 让拖动有顺滑的惯性
// 然后在动画循环里加一行 controls.update();
就这样,用户就能用鼠标 360° 拖看你的物体了。你看,从"一个方块"到"能交互的展示",中间只隔了几行。这就是 Three.js 把难活打包好的威力。

🆘 3D 特有的卡壳,和怎么破

🔍 深挖：3D 最常见的三种"看不到东西"

① 屏幕全黑/灰 → 多半没加光,或灯太弱。(第4站)先加一盏环境光+一盏平行光试试。

② 啥都没有/空白 → 多半相机没对准、或物体在相机背后。把相机往后退(调大 camera.position.z),或确认物体加进场景了。

③ 物体黑乎乎没立体感 → 材质和光不匹配。有些材质(如 MeshStandardMaterial)必须有光才显示;调试时可先用"不需要光的材质"(MeshBasicMaterial)排除问题。

排错方法和别的岛一样:把现象 + 控制台报错截图喂给 AI。3D 报错常在浏览器控制台(F12),WebGL 相关的提示也在那。

⚠️ 性能提醒:3D 比普通网页吃性能得多(每秒重画几十次、还有大量三角形)。手机上尤其要省着用。原则:三角形别太多、灯别太多、阴影按需开。做炫酷效果时让 AI 帮你注意"在手机上会不会卡"。

⭐ 3D 视觉岛总纲

3D 是用明暗远近"骗"出的立体;任何 3D 画面都由五件套(场景🌌·相机📷·物体🧊·材质🎨·光💡)搭成。

物体本质是三角形网格;没光就看不见立体;动画靠每帧重画的循环。

🌍整个 3D 世界的版图

前面 8 站讲的是"网页 3D"这一条路。但 3D 是个很大的世界,你常听到的那些词(Blender、Unity、虚幻引擎、C4D……)其实分属四条不同的路。先把版图摊开,你就不会再被一堆名字绕晕。

🔍 深挖：先分清两个根本不同的动作——"造"和"用"

3D 世界所有事,可以先劈成两半:① 造出 3D 物体(建模)　② 把 3D 物体用起来(渲染/交互/做成产品)。

这就像"做家具" vs "用家具布置房间"是两个工种。很多新手的混乱,是把"造模型的软件"和"用模型的引擎"当成一回事。它们经常是不同的工具,接力配合(下一站的流水线会讲)。

记住这条分界,下面四条路你一看就懂它各自站在哪。

🌐 路一 · 网页 3D(你前 8 站学的)

用 Three.js 这类,让 3D 跑在浏览器里。优点:打开网页就能看、不用装、和你的 web 技能一脉相承。适合产品展示、网页装饰、轻量交互。这是对你最快出成果、最该先掌握的一条。

🎮 路二 · 游戏引擎(Unity / 虚幻 Unreal)

专门做游戏、虚拟世界、复杂实时交互的"全套工作室"。它把物理碰撞、角色控制、特效、声音、关卡全打包了。能做手机/电脑游戏、也能做 VR、建筑漫游。代价:比网页 3D 重得多,要学引擎本身,出成果慢。

🎬 路三 · 影视级渲染(离线渲染)

追求极致画质的静态图或动画片——电影特效、产品广告大片、建筑效果图。它不要求实时,可以一帧算几分钟甚至几小时,换来以假乱真的光影。和网页/游戏的"实时"是两种哲学(下面对比)。

🧱 路四 · 3D 建模(Blender 等)

专门"造"3D 模型的软件——把一个角色、一辆车、一栋楼"雕"出来。它造出的模型,再拿去喂给前面三条路用。Blender 免费且强大,是目前最值得新手入门建模的工具。

🔍 深挖：一个关键分水岭——"实时" vs "离线"渲染

实时渲染(网页3D、游戏):必须每秒画几十帧让你能即时交互,所以为了快,画质要做取舍。你能拖、能玩。

离线渲染(影视级):不用即时,一帧慢慢算到极致逼真,算完导出成图片/视频。你只能看成品,不能交互。

同样是 3D,这两种在"为快还是为美"上走了相反方向。理解这条,你就懂了为什么游戏画面再好也不如电影 CG 精细——不是技术不行,是游戏必须实时,没时间慢慢算。

🔗一个模型从"被造"到"进你产品"的流水线

新手常以为"做个 3D"是一个软件搞定的事。其实在专业流程里,一个 3D 物体往往要几个工具接力。看懂这条流水线,你就知道每个工具卡在哪个环节、自己该学哪段。

🔗 一个 3D 模型的一生(以"放进你网页"为例)
1🧱建模:在 Blender 里把形状"雕"出来(一堆三角形,第3站)
↓
2🎨材质/贴图:给它穿"皮肤"——颜色、纹理照片、金属/粗糙感(第2站材质)
↓
3🦴(可选)绑骨骼+动画:如果要它动(走路、挥手),给它装"骨架"再做动作
↓
4📦导出:存成通用格式文件(网页常用 .glb / .gltf)
↓
5🌐载入引擎/网页:用 Three.js 把这个 .glb 加载进来,配上光和相机(第6站),它就出现在你产品里了
🔍 深挖：为什么要"导出成通用格式"这一步

因为"造模型的软件"和"用模型的引擎"是分开的(第9站那条分界)。Blender 造好的模型,要让 Three.js 能读,得先翻译成一种双方都认的通用文件格式——网页世界最通用的就是 .glb / .gltf(你可以理解成"3D 界的标准打包格式")。

这和第一岛"前后端用 JSON 这种通用格式传话"是一个道理:不同工具之间协作,需要一种大家都认的中间格式。看到 .glb 文件别懵,那就是一个打包好的 3D 模型。

💡 对你这种 AI native 路线的好消息:你不一定要自己从头建模。网上有大量现成的 .glb 模型(免费/付费的模型库),你下载下来,直接让 Codex/Claude Code 帮你用 Three.js 加载进网页即可。"我有一个 .glb 模型文件,请用 Three.js 把它加载进网页,加好光照和鼠标拖动"——这种活它很熟。先用现成模型跑通,比一上来啃 Blender 快得多。

🥽AR / VR 和 AI 生成 3D

系统了解 3D,这两个热点绕不开。用大白话讲清它们是什么、和你前面学的什么关系。

🥽 AR 与 VR:把 3D "放进现实"或"包住你"

🔍 深挖：它们本质还是 3D 渲染,只是"看的方式"变了

VR(虚拟现实):戴上头显,你被一个 3D 世界完全包住,转头能看四周。本质是——把"相机📷"绑在你头上,你头转,相机就转(还记得第7站"你看到什么由相机决定"吗?这里到极致了)。

AR(增强现实):通过手机/眼镜,把 3D 物体叠加到你看到的真实画面上(比如在客厅里"摆"一个虚拟沙发看效果)。本质是——让 3D 物体的光影、角度,跟真实环境对齐,骗你觉得它真在那。

关键:它们底层还是你学的那套 3D 渲染(五件套、网格、光照),只是显示和交互的方式换成了头显/摄像头。所以你这一岛的地基,对 AR/VR 同样管用。网页里甚至有 WebXR 能让 Three.js 直接做 AR/VR。

🤖 AI 生成 3D:用一句话/一张图"变"出模型

🔍 深挖：它在帮你跳过最难的"建模"那一步

建模(第10站第1步)是 3D 里最费功夫的环节之一。现在有 AI 工具能根据一段文字描述、或一张图片,直接生成一个 3D 模型(.glb 那种)。

这对你这种"有 idea、想快速出活"的人意义很大:你可能不用学 Blender,就能让 AI 生成模型,再用 Three.js 放进网页。它把流水线最重的"造"那一步,变成了"描述一下"。

但要诚实:目前 AI 生成的 3D,质量和可控性还在快速发展中,精细产品级的东西未必够用,但做原型、做不太苛刻的展示已经很能打。这正是手册第七岛说的——AI 在帮你跳过"手上的功夫",但你仍需懂原理来判断生成的东西好不好、能不能用。

💡 怎么试:你可以让 ChatGPT/Claude 帮你梳理当前有哪些"文字/图片转3D"的工具(这类工具更新很快,以最新的为准),生成模型后,同样用第10站的方式让 AI 帮你加载进网页。原理你已经懂了,工具只是手段。

🎇经典代码效果案例

这一站全是"一看就哇"的经典效果。每一个都做成了你能拖动的真实 demo,并讲清它背后的那点核心思路——你会发现,炫酷效果的底子,往往是很朴素的小招式重复出来的。

✨ 案例一 · 粒子星空(3D 的"炫技 Hello World")

🔍 它的核心思路:成千上万个小点,各就各位

满屏流动的星辰,看着复杂,其实就是——生成几千个小点,每个点给一个随机的 (x,y,z) 位置(第3站说的坐标),让它们一起缓慢旋转。

关键招式叫 粒子系统(Particles):不去一个个造小球(太吃性能),而是把几千个点当成"一坨"一次性画出来。星空、烟雾、火花、下雪,底层都是它。数量堆出震撼,这是粒子的灵魂。

🌊 案例二 · 波浪网格(用数学让平面"活"起来)

🔍 它的核心思路:让每个顶点按正弦波上下起伏

第3站说物体是顶点拼的网格。波浪效果就是——拿一张平整的网格,在动画循环里,让每个顶点的高度按一个"波浪数学公式"(正弦 sin)随时间起伏。

你不用懂这个公式,只要知道:sin 是一个"平滑地一上一下循环"的函数,把顶点的高低交给它,平面就变成了起伏的水面/绸缎/山丘。这是"用数学驱动视觉"的最经典入门例子。

🔆 案例三 · 几何阵列 + 自发光

🔍 它的核心思路:复制 + 节奏 + 发光材质

一排会呼吸/律动的方块,思路是:用循环复制出一排物体(第5站的循环不只能用在时间上,也能用来"批量生成"),再让它们的高度/亮度按各自的位置错开节奏地变化——就有了"波浪式律动"。

"发光感"则来自自发光材质:普通材质要靠灯照才亮,自发光材质自己发光、不依赖灯(像霓虹灯管)。这是做赛博朋克、科技感界面的常用招。

🔆 律动方块阵拖动旋转

🖱️ 案例四 · 鼠标视差跟随

🔍 它的核心思路:相机/物体随鼠标轻轻偏移

很多高级网站,鼠标一动,画面里的东西会轻轻地跟着偏一点,产生"近的动得多、远的动得少"的视差感,立刻显高级。

原理朴素到你想不到:读出鼠标在屏幕上的位置,把它换算成一个很小的偏移量,每帧让相机(或物体)朝那个方向挪一点点。近大远小(第14站透视)会自动让近处的东西偏移看起来更明显——视差就出来了。

这四个"哇"效果,拆开都很朴素:粒子=很多点、波浪=顶点按sin起伏、阵列=复制+错开节奏、视差=随鼠标微偏。

炫酷往往不是某个高深魔法,而是简单招式 × 数量 × 节奏。看懂这点,你就敢让 AI 帮你做任何效果了——你能说清要什么。

🎨 美术篇 · 让 3D "好看"的,是这些画画的人早就懂的道理

📐透视：近大远小的灭点魔法

第1站说立体感靠"近大远小"。这件事在美术里有个精确的体系,叫 透视(Perspective)——画家几百年前就摸透了,3D 程序里它变成了相机的一个参数。

🔍 深挖：所有平行线,都奔向远处一个"灭点"

站在笔直的马路中间往远看:两条本来平行的路边,会在远方汇聚到一个点——这个点叫 灭点(Vanishing Point)。这是透视最核心的现象。

一点透视:只有一个灭点(比如正对着一条走廊)。两点透视:看一个立方体的角,左右两组线各奔一个灭点(最常见的"立体感"构图)。三点透视:再加上下方向(仰视高楼/俯视峡谷)。

在 3D 程序里,这一切由相机的 视野角度(FOV) 控制——它就是"镜头的广角程度"。FOV 大=广角=透视夸张(近大远小很猛,有冲击力但会变形);FOV 小=长焦=透视平缓(更稳、更接近平面)。这是你调"画面感觉"的关键旋钮。

下面这条"无限延伸的柱子走廊",你可以拖动滑块改变相机的 FOV(广角程度),亲眼看透视如何随之夸张或平缓——这就是导演选镜头、3D 里调 FOV 的同一件事:

📐 透视与灭点(拖滑块改广角)↓ 拖动滑块
广角 FOV: 55°
💡 一句能用上的:想要画面更有冲击力、戏剧化(产品大特写、英雄感)→ 用大 FOV(广角);想要画面稳重、真实、不变形(展示精细产品)→ 用小 FOV。这和摄影师选镜头是完全一样的判断。

💡光影：三件套让物体"立"起来

第4站讲了"没光看不见"。这一站讲美术上更进一层的:光打在物体上,会形成几个固定的区域,画准了它们,东西就"立体得可信"。素描课第一节教的就是这个。

🔍 深挖：一个球上的"五大调子"(记住前三个就够)

拿一个被一盏灯照着的球,它表面从亮到暗,有几个区域:

① 亮面/高光:正对着光的地方最亮,最亮的那个小点叫"高光"(物体越光滑高光越锐)。
② 灰面/过渡:从亮慢慢转暗的中间地带,这层渐变是"圆"的关键。
③ 暗面:背光的一面。
④ 反光:暗面里因为环境反射,会有一点点回亮(别让暗面死黑)。
⑤ 投影:物体在地面投下的影子——它把物体"钉"在了空间里,没有投影会像漂浮的贴纸。

在 Three.js 里:高光由材质的"光滑度"决定、明暗渐变由平行光方向决定、暗面别死黑靠加一点环境光、投影要专门打开阴影开关。美术规律 → 程序参数,一一对应。

⭐ 让东西"立体得可信"的口诀

亮面、暗面、投影——三件套缺一,立体感就塌。

暗面别死黑(加点环境光当反光);一定要有投影(把物体钉在地上)。

新手做的 3D "看着假",十有八九是光太平(没明暗渐变)或没投影。

🎨色彩：冷暖与对比决定质感

最后一块美术拼图:颜色。同一个形状、同样的光,换一套配色,质感和情绪天差地别。这也是有规律可循的,不是凭感觉乱调。

🔍 深挖：三条最实用的色彩规律

① 冷暖对比制造立体与氛围。美术里常用"暖色光打亮面、冷色光打暗面"(比如橙色主光 + 蓝色环境光)——因为现实中阳光偏暖、阴影偏冷。这一招能让画面瞬间"高级、有电影感"。你看大片海报几乎都是橙蓝对撞。

② 对比度决定"清晰"还是"柔和"。亮暗差距大=强对比=硬朗、戏剧化;差距小=低对比=柔和、高级、安静。不是越鲜艳越好,克制的对比往往更耐看。

③ 少即是多:主色 + 点缀。新手爱用很多种鲜艳颜色,结果很"廉价"。专业做法是定一个主色调,再用一两个点缀色提神。这和你前面网页岛说的"配色统一"是同一个道理。

⭐ 美术篇总纲

让 3D 好看,靠的是画家早懂的三件事:透视📐(灭点/FOV)、光影💡(亮暗+投影)、色彩🎨(冷暖/对比/克制)。

它们全都能翻译成 Three.js 的参数:FOV、光的方向、材质光滑度、阴影开关、颜色值。

所以你这个"导演"的活,不只是技术——懂一点美术规律,你才能判断"做得好不好看",并说清让 AI 怎么改。技术让它"动起来",美术让它"好看",两者都懂,才是完整的 3D 视觉。
